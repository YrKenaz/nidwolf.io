## 개요

경로: `ladder.html` (~1280줄, 단일 파일)
업무 중 팀원 추첨용 사다리타기 게임. Canvas 기반, 스킨 시스템 분리.

**핵심 제약**: 결과는 시작 전 1:1로 확정. 시각 연출만 극적으로 변화.

## 클래스 목록

### `SkinBase` (추상 클래스)

```javascript
class SkinBase {
  drawBg()
  drawRail(x, y1, y2)
  drawBar(x1, y1, x2, y2, alpha, color)   // 대각선 바 지원
  drawMarker(cx, cy, idx, n, lbl, op)
  drawLabel(x, y, txt, pos, idx)
  drawParticle(cx, cy, color, r, a)
  _cv(varName)                             // CSS 변수 읽기
}
```

### `DefaultSkin extends SkinBase`

현재 유일한 스킨. CSS 변수 연동 (다크/라이트 자동 반영).
마커 색상 10가지 순환: `#7C6CF5` `#3EC9A7` `#F59E0B` `#E24B4A` …

**새 스킨 추가 방법**:
```javascript
class RPGSkin extends SkinBase { /* ... */ }
SM.reg('rpg', RPGSkin, 'RPG 도트');
```

### `Engine` (LadderEngine)

```javascript
engine.trueBars   // normal | hidden 바 (경로 영향 O)
engine.trapBars   // trap 바 (경로 영향 X, 가짜)
engine.allBars    // trueBars + trapBars
engine.paths[]    // 각 참가자 waypoints (논리 좌표)
engine.mapping[]  // mapping[i] = 결과 컬럼 인덱스
engine.portals[]  // 포탈 쌍 [{entry, exit}]

engine.trigger(bar)     // 바 페이드 애니메이션 시작
engine.tickBars(dt)     // 매 프레임 alpha 전환
engine.tickPortals(dt)  // 포탈 회전 애니메이션
```

### `AnimCtrl` (AnimationController)

```javascript
anim.cidx              // 현재 플레이어 인덱스
anim._trapRevealSet    // 현재 플레이어 경로 위 trap 바 Set
anim._teleport         // {phase, t, dur, opacity, toX, toY} | null

anim.init(cfg)
anim.startCurrent()    // cidx 플레이어 시작, _trapRevealSet 재계산
anim.setSpeed(s)       // 0.6 / 1 / 2
anim.burst(cx,cy,colors,n)
anim.onDone = (playerIdx) => {}
```

## 바(Bar) 동적 시스템

| 타입 | 초기 alpha | 트리거 시 | 경로 영향 | 색상 |
|------|-----------|-----------|-----------|------|
| `normal` | 1 (보임) | 변화 없음 | ✅ | `--cv-bar` |
| `hidden` | 0 (숨김) | 페이드 인 (teal) | ✅ | `#3EC9A7` |
| `trap` | 1 (보임) | 페이드 아웃 (red) | ❌ | `#E24B4A` |

### 트리거 파이프라인

```
_update() 루프
  → onBar = (markerCol === bar.col || markerCol === bar.col + 1)
  → if !onBar → skip
  → barTrigY = (markerCol === bar.col) ? by1 : by2   ← 컬럼별 Y
  → if |m.y - barTrigY| > TRIGGER_PX(72) → skip
  → if trap && !_trapRevealSet.has(bar) → skip
  → eng.trigger(bar)
```

### `_trapRevealSet` 계산 (`startCurrent`)

```javascript
// 플레이어 경로에서 수직 하강 구간 추출
descendedThrough = Set of "col:row"

// trap 바: 왼쪽 OR 오른쪽 레일 모두 체크 (대각선 바 대응)
_trapRevealSet = trapBars.filter(b =>
  descendedThrough.has(`${b.col}:${b.row}`) ||       // 왼쪽 레일
  descendedThrough.has(`${b.col+1}:${b.rowEnd}`)     // 오른쪽 레일 (대각선)
)
```

> ⚠️ **버그 이력** (BUG-003): 초기 구현은 왼쪽 레일만 체크 → 대각선 바의 오른쪽 레일로 통과 시 트리거 불가. `2026-06-20` 수정.

## 포탈 시스템

waypoint 타입: `portal_entry` → `portal_exit`

```javascript
// AnimCtrl 텔레포트 상태
_teleport = { phase: 'in'|'out', t, dur, opacity, toX, toY }

// 'in' 페이즈: 마커 투명화 → m.wpIdx를 portal_exit로 이동
// 'out' 페이즈: 도착 좌표에서 마커 출현, m.t = 0 으로 리셋
//   → portal_exit에서 정상 이동 시작 (wpIdx 추가 증가 없음)
```

> ⚠️ **버그 이력** (BUG-002): 'out' 페이즈 종료 시 `m.wpIdx++` 중복 실행 → 포탈 직후 다음 이벤트로 점프. `2026-06-20` 수정.

## 맵 생성 리빌 애니메이션 (`_revealMap`)

게임 시작 버튼 클릭 후 2초간 맵 생성 과정을 보여준 뒤 플레이 시작.

```
Phase 1 (raw 0.0 → 0.55): 레일 위→아래 클립 성장
Phase 2 (raw 0.25 → 1.0): 바 row 순으로 점진 출현
  - hidden 바: 표시 안 함      (정체 숨김)
  - trap 바:   일반 바 색 표시 (정체 숨김)
  - normal 바: 그대로 표시
easeInOutQuad 타이밍 함수 적용
```

## 플레이 흐름

```
시작 버튼 클릭
  → Engine.build() : 경로 사전계산, 바·포탈 배치
  → _revealMap() : 2초 애니메이션
      → onDone : 바 alpha 초기화 (normal=1, trap=1, hidden=0)
  → _goNext(0)
      → anim.cidx = 0 → startCurrent() + _trapRevealSet 구성
      → 마커 이동 (420 px/s 기본)
      → 72% 도달 → 슬로우모션
      → 완료 → onDone(0) → _goNext(1) → …
      → 마지막 완료 → 가짜 당첨 → "오류!" → 실제 결과 표시
```

## 대형 파일 수정 시 주의사항

> ⚠️ Edit 도구는 ~1280줄 파일에서 묵시적으로 잘릴 수 있음. Python `str.replace()` 패치 스크립트 사용할 것.

```python
path = '/sessions/.../NoNameProject/ladder.html'
txt = open(path, encoding='utf-8').read()
assert 'OLD_PATTERN' in txt, '패턴 없음'
txt = txt.replace('OLD_PATTERN', 'NEW_PATTERN', 1)
open(path, 'w', encoding='utf-8').write(txt)
```
