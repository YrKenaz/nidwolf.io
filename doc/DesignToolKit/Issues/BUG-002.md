# [BUG-002] 포탈 이동 후 마커가 다음 이벤트 위치로 순간이동

← [[_Index]] | [[🏠 Home]]

## 요약

> 포탈 텔레포트 'out' 페이즈 종료 시 `m.wpIdx`가 중복 증가하여, 마커가 portal_exit 직후 waypoint 위치로 점프함.

## 환경 / 재현 조건

- 파일: `ladder.html` — `AnimCtrl._updateTeleport()`
- 발생 조건: 사다리 경로에 포탈이 포함된 경우

## 증상

- 포탈 도착 애니메이션 완료 직후 마커가 portal_exit 위치가 아닌 그 다음 바/종착점으로 순간이동
- 포탈 이후 구간의 바 트리거·슬로우모션이 스킵됨

## 원인 분석

텔레포트 2단계 구조:
- **'in' 페이즈**: 마커 투명화 → 마지막에 `m.wpIdx`를 portal_exit 인덱스로 이동
- **'out' 페이즈**: portal_exit 좌표에서 마커 출현

문제는 'out' 페이즈 종료 코드에 `m.wpIdx++`가 하나 더 있었음:
```javascript
// 수정 전 — 잘못된 코드
tp.opacity = 1;
this._teleport = null;
m.wpIdx++;   // ← 이미 portal_exit를 가리키는데 한 번 더 증가!
m.t = 0;
```

## 수정 내용

'out' 페이즈 종료에서 `m.wpIdx++` 제거, `m.t = 0`만 수행:

```javascript
// 수정 후
tp.opacity = 1;
this._teleport = null;
// m.wpIdx는 'in' 페이즈에서 이미 portal_exit를 가리킴
m.t = 0;   // portal_exit 에서 정상 이동 시작
```

## 검증

- [x] 포탈 도착 후 portal_exit 위치에서 마커가 정상 출현
- [x] portal_exit 이후 바 트리거·슬로우모션 정상 작동
