# [BUG-001] build.py f-string 중간 절단 → 아티클 생성 오류

← [[_Index]] | [[🏠 Home]]

## 요약

> `build.py` 내 f-string 274번째 줄이 잘려 `{len(HIST_ALL)} articles</small></span></h2>` 부분이 누락, 아티클 index.html 생성 실패.

## 환경 / 재현 조건

- 파일: `_dev/_gen/build.py`
- 발생 조건: `python build.py` 실행 시

## 증상

- `articles/index.html` 생성 시 아티클 수 표기 부분이 잘린 채 출력
- 이후 라인의 `gen_static_pages` 호출부도 가비지 문자로 깨짐

## 원인 분석

Edit 도구가 대형 파일을 패치할 때 파일을 묵시적으로 잘라냄.  
→ f-string 274줄 이후 내용이 유실됨.  
→ `gen_static_pages` 호출부 3줄도 heredoc 처리 중 인코딩 깨짐.

## 수정 내용

1. Python 스크립트로 파일을 읽어 절단 지점 확인
2. 완전한 `{len(HIST_ALL)} articles</small></span></h2>` 라인으로 교체
3. 깨진 `gen_static_pages` 호출부 3줄 제거 (about/privacy는 수동 유지)

```python
# 패치 방식
src = open(BUILD, encoding='utf-8').read()
src = src.replace('OLD_TRUNCATED', 'CORRECT_LINE', 1)
open(BUILD, 'w', encoding='utf-8').write(src)
```

## 검증

- [x] `python build.py` 정상 완료 확인
- [x] `articles/index.html` 아티클 수 표기 정상

## 교훈

> ⚠️ 대형 파일(build.py, design-toolkit.html, ladder.html) 수정 시 **Edit 도구 금지** — Python `str.replace()` 패치 스크립트 사용.
