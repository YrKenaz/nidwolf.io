# [BUG-003] 함정/숨김 바를 밟아도 사라지거나 나타나지 않음

← [[_Index]] | [[🏠 Home]]

## 요약

> `_trapRevealSet` 필터가 대각선 바의 왼쪽 레일만 체크하여, 플레이어가 오른쪽 레일로 통과할 경우 trap/hidden 바 트리거가 차단됨.

## 환경 / 재현 조건

- 파일: `ladder.html` — `AnimCtrl.startCurrent()` / `AnimCtrl._update()`
- 발생 조건: 대각선 바(rowDelta > 0)의 오른쪽 레일(`bar.col+1`)로 마커가 통과할 때

## 증상

- trap 바(함정): 마커가 위를 지나가도 사라지지 않음
- hidden 바(숨김): 마커가 위를 지나가도 나타나지 않음 (일부 케이스)

## 원인 분석

### 원인 1 — `_trapRevealSet` 왼쪽 레일만 체크 (주 원인)

```javascript
// 수정 전 — 왼쪽 레일(bar.col:bar.row)만 체크
this._trapRevealSet = new Set(
  eng.trapBars.filter(b => descendedThrough.has(`${b.col}:${b.row}`))
);
```

`descendedThrough`는 플레이어가 수직 하강하는 `col:row` 집합.  
플레이어가 `bar.col+1`(오른쪽 레일)로 내려올 경우 집합에 `bar.col:bar.row`가 없음  
→ trap 가드 `if (!_trapRevealSet.has(bar)) return`에서 트리거 차단.

### 원인 2 — `barTrigY` 고정값 (부 원인)

```javascript
// 수정 전 — 항상 min(by1, by2) 사용
const barTrigY = Math.min(by1, by2);
```

대각선 바(rowDelta=2)에서 by2 - by1 ≈ 37.6 px.  
TRIGGER_PX=72 이내라 실제 영향은 미미하지만, 컬럼별 정확한 Y를 사용하는 것이 올바름.

## 수정 내용

### Fix 1 — `_trapRevealSet` 양끝 체크

```javascript
this._trapRevealSet = new Set(
  eng.trapBars.filter(b =>
    descendedThrough.has(`${b.col}:${b.row}`) ||        // 왼쪽 레일
    descendedThrough.has(`${b.col+1}:${b.rowEnd}`)      // 오른쪽 레일 (대각선)
  )
);
```

### Fix 2 — `barTrigY` 컬럼별 Y, `onBar` 체크를 앞으로 이동

```javascript
// 수정 후 — onBar 먼저 확인 후 컬럼별 Y 계산
const onBar = markerCol === bar.col || markerCol === bar.col + 1;
if (!onBar) return;
const barTrigY = (markerCol === bar.col) ? by1 : by2;
if (m.y < barTrigY - TRIGGER_PX) return;
if (m.y > barTrigY + TRIGGER_PX) return;
```
