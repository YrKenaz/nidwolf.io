# [BUG-004] build.py 한국어 print문 heredoc 인코딩 깨짐

← [[_Index]] | [[🏠 Home]]

## 요약

> build.py 하단 한국어 print 구문이 heredoc 처리 과정에서 UTF-8 바이트열이 손상되어 실행 시 UnicodeDecodeError 또는 깨진 문자 출력.

## 환경 / 재현 조건

- 파일: `_dev/_gen/build.py`
- 발생 조건: `python build.py` 실행 시 콘솔 출력 단계

## 증상

- 완료 메시지 print 구문에서 한국어가 `???` 또는 `\xec\x95\x84` 형태로 출력
- 일부 환경에서 `SyntaxError: Non-UTF-8 code` 발생

## 원인 분석

Edit 도구가 UTF-8 멀티바이트 한국어 문자를 heredoc 내에서 처리할 때 바이트 경계를 잘못 나눔.  
→ 파일 인코딩이 부분적으로 손상.

## 수정 내용

Python 스크립트로 파일의 손상 지점을 찾아 올바른 UTF-8 콘텐츠를 직접 append:

```python
# 손상 지점 이전까지 읽기
src = open(BUILD, 'rb').read()
cutoff = src.rfind(b'def gen_articles')   # 안전한 기준점
src_clean = src[:cutoff] + CORRECT_SUFFIX.encode('utf-8')
open(BUILD, 'wb').write(src_clean)
```

## 검증

- [x] `python build.py` UTF-8 출력 정상
- [x] `# -*- coding: utf-8 -*-` 선언 확인

## 교훈

> 한국어가 포함된 파일은 항상 `encoding='utf-8'`로 읽고 쓸 것.  
> Edit 도구 대신 Python `str.replace()` 패치 스크립트 사용.
